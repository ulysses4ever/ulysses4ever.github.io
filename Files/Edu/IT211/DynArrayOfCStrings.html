Проект «Динамический массив C-строк»

Мотивация. Реализацией абстрактного типа данных «динамический массив» в C++ можно считать шаблон класса std::vector. Будучи шаблоном, он позволяет получать массивы различных типов, однако для типа элемента подразумевается семантика значения (value semantics). Это значит, что, например, при выполнении метода push_back в вектор добавляется копия элемента. Такая семантика может оказаться неудовлетворительной для типов, не обладающих семантикой значений, например, для C-строк.

C-строка имеет тип char *. При добавлении значения такого типа в вектор создаётся копия указателя, но не копия строки, что может быть нежелательным и приводить к ошибкам. Стандартным решением такой проблемы является написание класса-обёртки для «плохого» типа, в нашем примере — для C-строки. Такой обёрткой в стандартной библиотеке C++ можно считать тип std::string и в большинстве случаев стоит использовать его (и соответственно, vector<string> для динамического массива строк). Однако на практике возникает необходимость работать с C-строками: например, при использовании библиотек, написанных на чистом языке C. В этом случае есть два выхода: (1) всё время проводить преобразования C-строка → std::string / std::string→ C-строка, что может сильно снизить эффективность, либо (2) работать исключительно с C-строками. В случае (2) вместо использования std::vector для организации динамического массива C-строк стоит рассмотреть реализацию специализированного класса, представляющего динамический массив C-строк.
  1. (По мере создания класса (см. ниже) в основной программе проверяйте корректность добавляемых функций.)
  2. Создайте класс cstring_vector с полями:
    • data типа указатель на C-строку — для хранения массива C-строк;
    • size типа size_t — текущий (логический) размер массива строк;
    • capacity типа size_t — текущая ёмкость (физический размер массива data — размер выделенной для него памяти).
    (Всегда должен быть выполнен инвариант класса: size <= capacity.) Также объявите и инициализируйте две статические константы класса: MIN_CAPACITY = 5 (минимальная ёмкость) и CAPACITY_FACTOR = 2 (множитель для увеличения ёмкости при её нехватке). Создайте геттер для поля size.
  3. Создайте приватную функцию-член
    void init_from_cstr_array(char ** cstr_array, size_t size);
    для инициализации data из данного массива C-строк данной длины. Алгоритм:
    • выделяется память для массива data размера size;
    • в цикле для каждого элемента data выделяется память, достаточная для хранения соответствующей строки из cstr_array (нужно узнать её длину — strlen — и не забыть про дополнительное место для завершающего нулевого байта) и копируется эта строка (strcpy).
    В этой же функции задайте нужные значения для полей size и capacity.
  4. Для инициализации всех трёх полей класса определите конструкторы cstring_vector:
    • без параметров, который выделяет для массива data память под MIN_CAPACITY C-строк, инициализирует size нулём и capacity значением MIN_CAPACITY.
    • с двумя параметрами char ** cstr_array и size_t size, для инициализации из массива C-строк; используйте init_from_cstr_array.
    • с двумя параметрами char const ** cstr_array и size_t size, для инициализации из массива константных C-строк; используйте init_from_cstr_array вместе со снятием константности с C-строк с помощью оператора const_cast<?>(cstr_array) (вместо ? должен стоять тип, к которому приводится cstr_array).
    • с одним параметром std::vector<std::string>. При копировании std::string в C-строки удобно пользоваться функцией-членом c_str() и стандартной функцией копирования C-строк strcpy.
    Проверьте работу всех конструкторов. Конструктор из константных C-строк должен проверяться с помощью переменной
    char const * consts_cstr_arr[] = {"Anything", "you", "like"};
    Помните, что вариант без квалификатора const в таком объявлении потенциально ошибочен (компилятор GCC, например, в этом случае выдаст предупреждение; компилятор Visual C++ в Debug-сборке сгенерирует код таким образом, что при попытке модификации любой строки массива будет выдана ошибка времени выполнения; проверьте это на вашем компиляторе).
  5. Определите приватную функцию-член deallocate, которая сначала в цикле освобождает память от каждой C-строки в массиве data, а затем память самого массива data, выставляя при этом size и capacity в 0. Оператор освобождения памяти из-под массива (в том числе, C-строки):
    delete [] p;
    где p — указатель на соответствующий массив.
  6. Определите деструктор класса cstring_vector, который вызывает deallocate.
  7. Определите конструктор копий и операцию копирующего присваивания. Используйте функции init_from_cstr_array и deallocate.
  8. Определите операцию обращения по индексу, возвращающую соответствующий элемент массива data, то есть указатель на char. Добавьте константную версию этой операции, возвращающую указатель на константу типа char. Вспомните, что в классе my_vector, продемонстрированном на лекции, возвращалась ссылка на хранимый объект массива. В комментариях к этой паре функций перечислите все следствия такого отличия (возможность менять строки: их адреса, их длину, их отдельные символы).
  9. Реализуйте операцию push_back, которая добавляет копию полученной C-строки в вектор. Если размер вектора достиг его ёмкости, необходимо перераспределить память (для этого стоит завести отдельную закрытую функцию-член): выделяется новый динамический массив указателей на char (new_data), размер которого в CAPACITY_FACTOR раз больше текущего размера, первые size его элементов заполняются значениями из массива data, массив data удаляется, переменной data присваивается значение new_data. Не забудьте также изменить значения полей size и capacity.
  10. Реализуйте функции-члены insert(i, str) и erase(i) для вставки/удаления C-строк в/из массива. В случае некорректного индекса, выбрасывайте исключение out_of_range (<stdexcept>).
  11. Реализуйте перегрузку операций сравнения на равенство и на неравенство объектов класса cstring_vector.
  12. Перегрузите операции вывода в поток и чтения из потока вектора C-строк. Ввод должен продолжаться до тех пор, пока не будет введена пустая строка. Продемонстрируйте их работу для объектов cin/cout, для файлов и для строковых потоков (см. предыдущее занятие, задача 12).
  13. Реализуйте метод sort, который сортирует строки массива лексикографически методом быстрой сортировки. Для этого воспользуйтесь имеющейся реализацией быстрой сортировки массива целых. Всё, что нужно: поменять в нужных местах исходный тип элементов (int) на используемый в нашем классе тип элементов (C-строки). Кроме того, вместо сравнения целых с помощью <, следует использовать стандартную функцию strcmp.
