Службы прикладного уровня

Упражнения
I. Передача файлов по протоколу FTP: клиент ftp.
  1. Выполните команду "ftp ftp.rsu.ru". В качестве имени пользователя введите "anonymous", а в качестве пароля свой адрес электронной почты.
  2. Определите текущий каталог ftp-сервера (pwd).
  3. Просмотрите содержимое этого каталога (ls).
  4. Перемещаясь по каталогам, найдите каталог с именем "games" (cd, ls).
  5. Завершите соединение с сервером ftp (quit).
  6. Находясь на сервере losfs, создайте в своем каталоге файл abc.dat, содержащий семь любых латинских букв.
  7. Подключитесь по ftp к серверу sun.mmf.rsu.ru и создайте в своем домашнем каталоге на этом сервере подкаталог Letters.
  8. Скопируйте файл abc.dat в созданный каталог.
  9. Скопируйте какой-нибудь файл с сервера на клиент.
  10. Завершите сеанс ftp.
II. Служба электронной почты: клиент mail, кодирование сообщений.
  1. Запустите клиент mail и просмотрите все сообщения.
  2. Удалите сообщение с номером 2.
  3. Сохраните первое сообщение в файле с именем letter1.
  4. Завершите работу клиента mail.
  5. Отправьте письмо пользователю bravit с темой "Lab2: sending mail" и телом, содержащим Ваши фамилию и имя.
  6. Подготовьте файл с сообщением для Вашего знакомого и отправьте его.
  7. Выполните кодирование любого двоичного файла и посмотрите результат этого кодирования, сравните размеры исходного и закодированного файлов.
  8. Осуществите декодирование файла и сравните результат с исходным файлом.
III. Протокол SMTP для передачи сообщений электронной почты.
  1. Подключитесь по SSH к серверу losfs.
  2. Выполните команду "telnet sun.mmf.rsu.ru 25" и дождитесь сообщения "220 sun.mmf.rsu.ru".
  3. Поприветствуйте сервер командой "HELO losfs.math.rsu.ru" ("HELO" - это не опечатка!").
  4. Ввведите команду "MAIL FROM: " (адрес может быть любым).
  5. Ввведите команду "RCPT TO: user" (user - это адрес получателя на сервере sun.mmf.rsu.ru, здесь нужно ввести свой личный логин).
  6. Введите команду "DATA".
  7. Теперь можно вводить текст сообщения (не забудьте о пустой строке, разделяющей заголовок и тело сообщения).
  8. После последней строки сообщения введите строку, содержащую точку: ".".
  9. Выполните команду "QUIT". На этом соединение закончится.
  10. Подключитесь к серверу sun по протоколу SSH, запустите клиент mail и убедитесь, что сообщение доставлено.
IV. Протокол доставки почты POP3.
  1. Попробуйте подключиться к своему pop3-серверу и посмотреть список хранящихся там сообщений (по желанию).
V. Схема работы службы WWW, клиенты HTTP.
  1. Загрузите в свой домашний каталог файл с этой лабораторной работой (wget www.highart.ru/students/cnet/lab2.html) и просмотрите его содержимое.
  2. Откройте ту же страницу в браузере lynx, попробуйте поперемещаться по ссылкам, перейдите на сайт uic.rsu.ru
  3. С помощью протокола FTP скачайте файл lab2.html из домашнего каталога на сервере sun на локальную машину и просмотрите его в каком-нибудь графическом браузере, сделайте выводы.
  4. Найдите какой-нибудь голосовой браузер (домашнее задание).
VI. Запросы и ответы в протоколе HTTP.
  1. Подключитесь к веб-серверу www.rsu.ru по протоколу HTTP ("telnet www.rsu.ru 80").
  2. Введите текст запроса для главной страницы. Не забудьте дважды нажать клавишу Enter в конце запроса.
  3. Исследуйте заголовки ответа сервера, попытайтесь определить их значение.
  4. Попробуйте послать серверу запрос на несуществующую страницу и проанализируйте результат такого запроса.
  5. Попробуйте послать серверу неправильный запрос (например, HELLO) и проанализируйте результат такого запроса.
VII. Введение в язык разметки гипертекста HTML.
  1. Напишите на локальном компьютере HTML-документ, содержащий следующую информацию о Вас: фамилия, имя, факультет, курс, группа, хобби и что-нибудь еще. Поставьте ссылки на Ваших одногруппников.
  2. Создайте в домашнем каталоге на сервере sun подкаталог "public_html".
  3. Скопируйте созданную страницу по протоколу FTP в каталог public_html на сервере.
  4. С помощью браузера lynx обратитесь к Вашей странице по адресу: sun.mmf.rsu.ru/~login/имя_файла (login - это Ваше имя на сервере sun).

Вспомогательные материалы
I. Передача файлов по протоколу FTP: клиент ftp.
Протокол FTP (file transfer protocol) предназначен для копирования файлов с одного компьютера на другой. При этом один из компьютеров является сервером (на нем должен быть запущен соответствующий "демон", напр. ftpd), а другой — клиентом. В роли клиента FTP могут выступать различные файловые менеджеры (Far, Total Commander) или специализированные программы. Во время взамодействия по протоколу FTP возможны операции в двух файловых системах: локальной (на комапьютере-клиенте) и удаленной (на сервере). Большинство клиентов поддерживают операции просмотра содержимого текущего каталога, смены текущего каталога, изменения прав доступа к файлам и каталогам, а также основные операции передачи и получения файлов. Следует иметь ввиду, что эти операции выполняются в двух файловых системах независимо.
Рассмотрим использование простого консольного ftp-клиента. Для подключения к некоторому серверу достаточно выполнить команду "ftp имя_сервера". Если подключение проходит успешно, то после этого требуется ввести имя пользователя на данном сервере и его пароль. Существуют так называемые анонимные ftp-сервера, которые позволяют подключаться любому пользователю. Они обычно не поддерживают возможность закачивания файла на сервер, но позволяют скачивать с сервера любые имеющиеся файлы. В случае входа на такой сервер нужно ввести имя пользователя "anonymous" и в качестве пароля указать свой адрес электронной почты.
После установления соединения клиент переходит в режим интерпретации команд пользователя. Вот некоторые команды клиента ftp:
"pwd" - имя текущего каталога на сервере
"ls" - вывод содержимого текущего каталога на сервере
"cd имя_каталога" - смена текущего каталога на сервере
"lpwd" - имя текущего каталога на клиенте (l - локально)
"lcd" - смена текущего каталога на клиенте
Непосредственно для передачи файлов используются команды put (закачивание на сервер) и get (скачивание с сервера). Эти команды можно использовать для передачи одного файла. При необходимости можно также воспользоваться командами mget и mput, которые поддерживают маски имен файлов. Примеры:
"put a.txt" - закачать на сервер файл a.txt (из текущего каталога на клиенте в текущий каталог на сервере)
"mput *.txt" - закачать на сервер все файлы из текущего каталога с расширением txt
"get 111.jpg" - скачать с сервера файл 111.jpg
"mget *.jpg" - скачать с сервера все файлы из текущего каталога с расширением jpg
При передаче текстовых файлов между компьютерами, работающими под разными операционными системами, может возникнуть проблема концов строк. Причина этой проблемы в том, что в разных системах используются разные символы для обозначения конца строки в текстовых файлах. Так, в DOS (и Windows) в качестве конца строки используется пара символов с кодами 13 и 10 (13 - CR - carriage return, 10 - LF - line feed), в Unix - один символ с кодом 10, а в MacOS - символ с кодом 13. Большинство клиентов ftp поддерживают автоматическое изменение символов конца строки в случае такой необходимости, но замена полезна только при передаче текстовых файлов, а в двоичных файлах она может привести к порче содержимого передаваемого файла. В связи с этим существуют два режима передачи файлов: ascii (для передачи текстовых файлов - c заменой символов конца строки) и binary (для передачи двоичных файлов - без преобразований). Необходимо следить за тем, в каком режиме находится клиент, и при необходимости переключаться между ними (командами ascii и binary соответственно).
Для завершения работы с клиентом ftp достаточно выполнить команду quit.
II. Служба электронной почты: клиент mail, кодирование сообщений.
Служба электронной почты предназначены для обмена текстовыми сообщениями. Каждый пользователь может иметь адрес электронной почты на некотором почтовом сервере. Этот адрес состоит из двух частей (имя пользователя и имя почтового сервера), разделенных символом "commercial at" (@ - "собака"). Например, name@server.ru - это адрес пользователя с именем "name" на сервере "server.ru". Для отправки сообщения необходимо знать адрес его получателя. Для получения почты необходимо знать имя пользователя и пароль на соответствующем почтовом сервере.
Рассмотрим работу простого консольного клиента mail. Его можно запустить командой "mail". Если сообщений для текущего пользователя нет, то этот клиент не запустится, так как при вызове без параметров он используется для чтения электронной почты. Если же сообщения есть, то показывается их перечень. Все сообщения пронумерованы, просмотреть каждое из них можно командой "t номер", опять отобразить спиcок - командой "h", удалить сообщение - "d номер", сохранить сообщение в некоторм файле - "s имя_файла". Все прочтенные, но не удаленные сообщения при завершении работы клиента удаляются из почтового ящика и сохраняются в специальном файле в домашнем каталоге пользователя.
Клиент mail можно также использовать для отправки почты. Для этого при ее запуске в качестве параметра следует указать адрес получателя (для получателей на том же сервере, что и отправитель достаточно написать логин без имени сервера): "mail bravit". Всякое электронное сообщение состоит из двух частей: заголовка и тела. Эти части отделяются друг от друга пустой строкой. Заголовок состоит из строк в следующем формате: "имя: значение". Например:
"Subject: problem with exam" - поле темы сообщения, оно обычно содержит краткое описание содержимого письма.
"CC: ivanov@ivan.ru, petrov@petr.ru" - поле, содержащее список других получателей (кроме того, что был указан в параметрах mail) этого письма (CC - carbon copy).
Клиент обычно автоматически добавляет поля "From" и "To" для обозначения отправителя и получателя. После ввода полей заголовка следует дважды нажать клавишу Enter (для создания пустой строки) и затем вводить текст сообщения. После ввода последней строки сообщения нужно нажать комбинацию клавиш Ctrl-D (символ конца файла).
Иногда бывает удобно подготовить текст сообщения в файле (вместе с заголовком и телом) и отправить его командой "mail адрес_получателя < имя_файла".
Протоколы службы электронной почты изначально проектировались только для передачи текстовых файлов ASCII, содержащих символы с кодами от 1 до 127. Поэтому перед отправкой двоичных файлов их следует закодировать. Для кодирования может использоваться команда "uudecode". Предположим, что следует отправить пользователю user1 файл с именем "a.gif". Для этого необходимо выполнить следующее:
"uuencode -o out.msg a.gif b.gif" - закодировать файл a.gif и сохранить результат в файле "out.msg" (он теперь может быть отправлен по электронной почте), последнее имя - это имя файла, которое он получит после раскодирования (в нашем примере b.gif)
"mail -s"File a.gif" user1 < out.msg" - отправка файла out.msg пользователю user1, с помощью параметра -s добавлена тема сообщения
Рассмотрим теперь действия получателя:
"mail" - запуск почтового клиента и просмотр списка пришедших сообщений; предположим, что нужное нам письмо имеет номер "5"
"t5" - просмотр сообщения, естественно, при этом просмотре полезной информации мы не получим, так как письмо закодировано"
"s5 file.msg" - сохранение письма в файле file.msg в текущем каталоге
"q" - выход из клиента mail
"uudecode file.msg" - обратное преобразование файла (декодирование), в результате которого в текущем каталоге появится файл b.gif
Следует иметь ввиду, что при отправке и получении сообщений с вложениями в графических клиентах (напр. MS Outlook или The Bat) производятся те же самые действия, только они скрываются от пользователя.
III. Протокол SMTP для передачи сообщений электронной почты.
Протокол SMTP (Simple mail transfer protocol) лежит в основе службы электронной почты. Он отвечает за пересылку сообщения от компьютера отправителя до почтового сервера отправителя и от почтового сервера отправителя до почтового сервера получателя. Этот протокол является текстовым, поэтому взаимодействие клиента и сервера легко смоделировать без применения специальных почтовых клиентов. Для этой цели можно воспользоваться программой telnet, которая позволяет вводить команды текстового протокола и получать ответы на эти команды от удаленного сервера. Команда протокола SMTP - это написанные заглавными буквами сокращения слов английского языка (HELO, QUIT, DATA), ответы - это строки, начинающиеся с кода ответа.
Предположим, что нам необходимо отправить сообщение пользователю bravit, зарегистрированному на сервере sun.mmf.rsu.ru (это сервер, к которому мы будем подключаться по протоколу SMTP). Для этого нужно выполнить следующее:
"telnet sun.mmf.rsu.ru 25" - подключение к удаленному серверу по протоколу SMTP (25 - это номер порта, на котором демон этого протокола ожидает подключения), необходимо дождаться окончания процесса подключения(появится строка, начинающаяся с числа 220)
"HELO losfs" - приветствуем сервер (параметр команды HELO - это имя компьютера, с которого осуществляется подключение)
"MAIL FROM: " - указываем адрес отправителя, можно указывать любой адрес (например адрес президента США или вообще не существующий - подумайте, почему борьба со спамом в настоящее время не приносит существенных результатов)
"RCPT TO: bravit" - указываем адрес получателя, он должен быть зарегистрирован на сервере, к которому мы подключились
"DATA" - команда с которой начинается передача текста сообщения
"Subject: Hello from me" - тема сообщения
"" - пустая строка, разделяющая заголовок и тело сообщения (здесь нужно просто нажимать клавишу Enter)
"Text" - текст тела сообщения
"." - точка в отдельной строке обозначает конец сообщения
"QUIT" - завершение подключения (вообще говоря, можно опять выполнять MAIL FROM и т.д.)
IV. Протокол доставки почты POP3.
Протокол POP3 (Post office protocol Version 3.0) предназначен для просмотра полученных на адрес пользователя сообщений электронной почты и их доставки с почтового сервера на компьютер клиента. Этот протокол поддерживает следующий набор команд:
"user имя_пользователя" - имя пользователя, зарегистрированного на почтовом сервере, к которому выполняется подключение
"pass пароль_пользователя" - пароль пользователя
"list" - вывести список сообщений, хранящихся на сервере (при этом выводятся номера сообщений и их размеры)
"retr номер" - вывести текст сообщения с указанным номером (retr - от слова retrieve)
"dele номер" - удалить с сервера сообщение с указанным номером
"quit" - завершить подключение
Ответы сервера в случае успеха начинаются с "+OK", а в случае ошибки — с "-ERR".
Подключится к POP3-серверу можно с помощью программы telnet (на порт 110), например:
"telnet pop3.aaanet.ru 110" - pop3.aaanet.ru - это имя почтового сервера для адресов @aaanet.ru
V. Схема работы службы WWW, веб-клиенты.
WWW - это сокращение от World Wide Web - так называется одна из служб Интернета, предоставляющая каждому пользователю доступ к специальным образом оформленным документам, хранящимся на серверах службы Web (на веб-серверах). Для этого доступа необходимо использовать специальные программы (веб-клиенты, браузеры). Веб-клиенты и веб-серверы взаимодействуют между собой по протоколу HTTP (Hyper Text Transfer Protocol), а документы, хранящиеся на серверах, пишутся на языке HTML (Hyper Text Markup Language - язык разметки гипертекста) и называются веб-страницами (или HTML-страницами). Совокупность веб-страниц называется веб-сайтом.
Схема работы службы WWW очень проста (предположим, что используется графический веб-клиент):
  1. Пользователь вводит адрес интересующего его документа (URL) в адресной строке браузера, обычно этот адрес состоит из имени сервера и имени документа на этом сервере.
  2. Браузер посылает запрос на указанный сервер с просьбой прислать соответствующий документ.
  3. Сервер получает запрос и в ответ возвращает текст нужного документа.
  4. Браузер отображает документ в соответствии с некоторыми правилами (определенными в языке HTML) и при необходимости запрашивает дополнительные файлы (например, картинки).
Рассмотрим примеры веб-клиентов. В самом простом случае веб-клиент должен сохранять запрошенный документ на компьютере пользователя. Примером такого клиента является программа wget:
"wget www.yandex.ru/vesna.html" - получение страницы vesna.html (при этом текст страницы будет сохранен в файле vesna.html)
"wget -O out.html www.rsu.ru" - получение главной страницы сайта РГУ (файл будет сохранен в файле out.html).
Обратите внимание на то, что если в последнем примере убрать параметр -О с именем выходного файла, то документ будет сохранен в файле index.html. Это имя файла, который будет передан сервером, если клиентом запрашивается имя каталога (в некоторых случаях это имя может быть другим).
Для получения и просмотра HTML-документа можно воспользоваться, например, текстовым веб-клиентом lynx:
"lynx www.stanford.edu" - получение и отображение главной страницы веб-сайта Стэнфордского университета.
В этой программе можно клавишами вверх/вниз перемещаться по гиперссылкам на странице, клавишей Enter - переходить на другие страницы. Нажав клавишу "G" в момент просмотра страницы, можно ввести новый адрес документа. Клавиша "Q" позволяет завершить работу программы.
В некоторых случаях для получения и просмотра веб-страниц пользуются также графическими (Microsoft Internet Explorer, Mozilla Firefox, Opera, Konqueror, Safari, Amaya и пр.) и голосовыми (т.е. произносящими вслух текст страниц) браузерами.
VI. Запросы и ответы в протоколе HTTP.
Протокол HTTP определяет форматы двух видов сообщений: запросов клиента и ответов сервера. Любое сообщение состоит из двух частей: заголовка и тела сообщения. Тело отделяется от заголовка пустой строкой. В запросе тело может отсутствовать, а в ответе оно обычно содержит текст возвращаемого HTML-документа. В заголовках выделяется первая строка (строка статуса для запросов и строка состояния для ответов), а все остальные строки состоят из имени поля и его значения, разделенных двоеточием. Поля обычно несут дополнительную информацию о запрошенной странице (напр. дата модификации, кодировка, язык и пр.).
Рассмотрим пример простого взаимодействия с HTTP-сервером с помощью клиента telnet.
"telnet www.rsu.ru 80" - подключение к удаленному серверу по протоколу HTTP (80 - это номер порта, на котором демон этого протокола ожидает подключения)
"GET  /  HTTP/1.1" - запрос главной страницы сервера по протоколу HTTP версии 1.1
"HOST:  www.rsu.ru" - указываем имя сервера
"" - пустая строка, разделяющая заголовок и тело сообщения (здесь нужно просто нажимать клавишу Enter)
Ответ на этот запрос начнется со следующих строк:
HTTP/1.1 200 OK
Date: Mon, 14 Feb 2005 07:40:04 GMT
Server: Apache/1.3.27 (Unix) mod_auth_pgsql/0.9.12 mod_ssl/2.8.12 OpenSSL/0.9.6e rus/PL30.16
Transfer-Encoding: chunked
Content-Type: text/html; charset=koi8-r
Vary: accept-charset, user-agent
Connection: Close

<html>
<head>
<title>Ростовский Государственный Университет</title>
<link type="text/css" rel="stylesheet" href="rsu.css">
В первой строке ответа указывается код ответа (в данном случае "200 OK"). Далее следуют поля заголовка ответа (текущее время и дата на сервере, программное обеспечение сервера, тип содержимого - текст в формате HTML и пр.). После пустой строки начинается HTML-документ запрошенной страницы.
VII. Введение в язык разметки гипертекста HTML.
Любой язык разметки предназначен для представления текстовой информации. При этом документ помимо этой информации обычно содержит специальные команды разметки, которые определяют способ представления (в частности, оформление и форматирование) соответствующих частей текста. В случае языка HTML эти команды разметки называются тегами. Теги могут ограничивать фрагмент документа, при этом различают открывающий и закрывающий теги, которые вместе с текстом между ними образуют элемент (контейнер) HTML. В следующем примере разметки текст "sample text" является содержимым элемента "b":
<b>sample text</b>
Как видно из примера, теги заключаются в угловые скобки, причем закрывающий тег начинается с символа "/" ("слэш"). Язык HTML определяет возможные последовательности тегов, правила их вложения друг в друга и варианты отображения содержимого документов.
Расмотрим простейший пример документа HTML:
<html>
<head>
  <title></title>
</head>
<body>
  <b>sample text</b>
</body>
</html>
Содержимое элемента "title" обычно отображается в заголовке окна браузера. Все содержимое элемента "body" показывается в окне браузера. Текст в элементе "b" отображается полужирным шрифтом. Вот значения некоторых других элементов:
"<u>подчеркнутый текст</u>"
"<i>курсив</i>"
"<h1>

Заголовок первого уровня

</h1>"
"<h2>

Заголовок второго уровня

</h2>"
"<br>" - переход на следующую строку
"<p>Абзац текста</p>" -
Некоторые элементы имеют атрибуты (дополнительные параметры), например:
"<font color='red'>Текст, написанный красным цветом</font>"
